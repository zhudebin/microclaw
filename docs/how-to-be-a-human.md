# How to Be a Human

> 一篇关于"让聊天机器人更像人"的随笔 —— 我们做到了哪些，又有哪些大概率永远在路上。
> （配套的逐项实现清单见 `docs/IMPLEMENTED.md`。）

## 开篇：到底是什么让聊天"像人"？

我们花了很多力气让 MicroClaw 变得"更像人"。但在写代码之前，得先回答一个更难的问题：
**一个人在跟你聊天时，到底具备什么特质？**

把它拆开看，会发现"像人"不是一种能力，而是**两层叠加**：

1. **表层 —— 怎么说话**：短、自然、有节奏、读得懂你的情绪和分寸。
2. **深层 —— 怎么做事**：遇到难题会找对的专长，能同时盯好几件事，像同事一样回来汇报。

最妙的是这两层的**反差**：一个很厉害的人，表面上跟你随便聊两句、很轻松；可你真要东西时，
他背后调动了一整套能力，然后回你一句"搞定了，结论是这个"。

再往里一层，是**人格稳定、情绪流动**：他还是那个他（性格不变），但此刻的语气会随对话起伏。

下面按这几层，讲讲我们做了什么、以及哪些还没做到。

---

## 一、表层：会"说话"

### 人的特质

真人聊天，90% 的回复是一两句话。会一次发好几条短消息（"等下" → "我看看" → "找到了"）。
会读你的情绪——你急他就直给，你沮丧他先共情。在群里懂分寸，没被叫到就不插嘴。不说废话。

### 我们实现了

- **默认简短、结论先行**：写进了 `SOUL.md` 的 Chat style 和系统提示词的 `Conversational style`。
- **一次回多条短消息**：复用 `send_message` 发前置气泡，最后一条作正式回复。
- **读情绪调语气**：`src/mood.rs` 用零成本启发式（中英双语、保守触发）识别 frustrated / urgent /
  sad / confused / grateful / excited / playful，注入 `<conversation_mood>` 提示语气。
- **群聊分寸**：群里注入"group etiquette"——被叫到才答、简短、不插别人的话。
- **一次问一个问题、去掉客套**。

### 还没做到（诚实地说）

- **真正的打字节奏**：人发消息有停顿、有"正在输入"的呼吸感。我们用气泡近似了，但还没有跨平台
  默认开启的"逐字流式 + 条间随机延迟"。它仍然是"想好了一次发出"，不是"边想边敲"。
- **读懂言外之意**：mood 是关键词级的、单轮的。真正的人能读出**反讽、阴阳怪气、欲言又止**，
  还能追踪一段对话里**情绪的起承转合**。我们目前读不出 sarcasm，也不建模跨轮的情绪弧线。
- **真·幽默**：SOUL 鼓励"a dry sense of humor"，但幽默是被提示出来的，不是在恰当时机**自发**
  涌现的。人会接梗、会自嘲、会在尴尬时打圆场——这些我们还做不到稳定。

---

## 二、内核：稳定人格 + 流动情绪

### 人的特质

你认得出"还是那个他"——性格是稳定的。但情绪会随对话起伏。他记得你、了解你的偏好，
也知道你喜欢被怎样对待。重要的事记得牢，鸡毛蒜皮的渐渐淡忘。

### 我们实现了

- **稳定人格**：`SOUL.md` 注入 `<soul>`，定义性格、价值观、工作风格。
- **流动情绪**：mood 层负责"此刻的语气"，与稳定人格分离——**SOUL 不变，语气随情境变**。
- **记得你 / 了解你**：三层记忆 + `USER.md` 连贯画像；reflector 还会显式抽取你的**沟通风格偏好**
  （语言、简短还是详细、正式还是随意），跨会话生效。
- **人类式遗忘**：记忆按 `confidence × 时间衰减`，PROFILE（身份类）免疫——记重要的、忘琐碎的。

### 还没做到

- **人格会成长**：真人的性格也会被关系慢慢塑造。我们的 SOUL 是**静态**的——它不会因为跟你聊久了
  而变得更懂你、更"对你这个人"。没有"自我的演化"。
- **关系的纵深**：人对"刚认识"和"老朋友"是不同的。我们没有建模**关系亲疏随时间加深**——
  对所有人、所有阶段，基本是同一套距离感。
- **记忆是"重构"的**：人的记忆会模糊、会脑补、会因情绪而扭曲（这恰恰很"人")。我们的记忆是
  忠实存储 + 衰减归档，干净，但少了那种"记错了但很真实"的人味。

---

## 三、深层：能干得像个专业同事

### 人的特质

强不在"什么都自己会"，而在**知道把对的事交给对的专长**，能**同时盯好几件事**，
还会**像同事一样主动汇报进展**，而不是闷头干完才出声。

### 我们实现了

- **专家团队**：`specialists.rs` —— 数学家、画师、研究员、程序员、写手、分析师……
  主代理把硬活路由给对的专家（`sessions_spawn(specialist=…)`），可无限扩展。
- **并发多任务 + 命名**：后台真并发，每条线有人话名字（`label`），`subagents_list` 一眼看全。
- **像同事一样汇报**：`report_progress` 中途同步 `📊 [任务] (60%): …`（带节流）；
  主动 standup（长任务定期 `🛰️`）；fan-in（一批活全完成出一条 `🧩` 总结）。
- **能用名字提起一件事**："看看竞品调研到哪了""把分析任务停了"——按 label 操作。

### 还没做到

- **专家之间真正协作**：现在是"主代理派活、各自干、回来汇总"。真团队会**互相讨论、交接、纠偏**
  ——研究员把发现丢给写手、写手回问一句。我们的专家之间还不会**直接对话**。
- **靠谱的 ETA 和"卡住"判断**：standup 会标"⚠️ 无进展"，但那是**启发式**（超时长 + 没更新）。
  人能说"这个我大概还要十分钟"——我们还给不出真正的时间估计。
- **从经验里学手艺**：同事会越做越顺、形成默契。我们的专家 persona 是**写死**的，
  不会因为"上次这么做效果好"而自我改进。

---

## 四、主动性：会关心，但不打扰

### 人的特质

久没联系，朋友会主动问一句。但主动是**有分寸**的——只在真有价值时开口，不会刷屏。
也会**自己察觉时机**：看到你提过的事有进展，会主动同步。

### 我们实现了

- **长沉默关怀**：`idle_checkin`——沉默超时后，让 agent **有价值才发、否则回 SKIP**（不投递）。
- **有分寸的主动**：所有"主动发消息"的功能（standup / fan-in / idle check-in）一律
  **默认关闭 + 强节流 + 可开关**——宁可少说，不要打扰。

### 还没做到（这是最难的一块）

- **真正的"内心想法"**：人是**一边听一边在脑子里冒念头**，攒够了表达欲才插一句
  （学术上对应 *Inner Thoughts* 框架）。我们的主动性是**定时轮询 + 规则门控**，
  不是"持续生成一串念头、择机发声"。它不会在一场多人热聊里**自然地抢到一个话茬**。
- **预测性的体贴**：人会提前想到"你明天有个会，要不要我帮你准备"。我们没有把日历、
  待办、承诺真正变成**主动触发源**——idle check-in 靠的是 LLM 临场判断，不是结构化的前瞻。
- **判断"此刻该不该说话"的细腻度**：人能感觉到"现在气氛不对，先别开玩笑"。
  我们的门控还很粗（沉默时长、是否被 @），读不出**群里的微妙气氛**。

---

## 五、还有些"很人"的东西，我们刻意没做

有些特质很像人，但对一个助手未必是好事，我们**有意识地没有去模拟**：

- **会忘事、会拖延、会情绪化地不回消息** —— 可靠性优先，这些"人味"我们不要。
- **为了显得厉害而不懂装懂** —— 我们让它**诚实地说"不确定，我查一下"**。
- **越界的亲密感** —— companion 类产品会刻意制造依赖；我们把"主动打扰"全部默认关闭，
  把分寸权交给用户。

**"像人"是手段，不是目的**。目的是：一个**好聊、靠谱、有分寸**的伙伴。

---

## 结语：一张把它串起来的画面

> 你随口问一句 → 它**短短**应一声；
> 真要硬活 → 它悄悄派**数学家 + 画师 + 研究员**几条线同时跑，
> 中途 `📊` 同步，完事 `🧩` 给你一条总结；
> 你沮丧时它**先共情再处理**，群里没叫它就**不插嘴**，
> 久没聊它**偶尔关心一句、但绝不刷屏**。
> 这一切之上，是一个**性格始终如一**的它。

我们做到了"表层会聊、深层能干、人格稳定、主动有度"。

而**还在路上的**，恰恰是最难、也最迷人的那部分：读懂言外之意、自发的幽默、
持续的内心念头、关系随时间加深、以及——**真正学会做它自己**。

也许 "How to be a human" 这件事，本来就没有终点。这正是它有意思的地方。
