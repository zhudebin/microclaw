# 总纲：如何让 MicroClaw 像一个人——甚至一个很专业的人

> 这是把 `docs/humanlike-chat-analysis.md`（拟人化）和
> `docs/concurrent-tasks-progress-reporting.md`（并发多任务）缝在一起的总纲，
> 回答一个问题：**到底怎么才能像一个人，甚至像一个很专业的人在跟你聊天？**

---

## 0. 核心洞察：「像人」是两层，关键在反差

大家常把"像人"只理解成"会聊天"。其实它是**两层**，缺一不可：

1. **表层 · 聊天的真实感**（怎么说话）——简短、一次发几条、有节奏、读空气、不啰嗦。
2. **深层 · 能力的真实感**（怎么做事）——像一个背后有整个团队的专业人士：
   会把难题分给对的专家、同时推进好几件事、像同事一样回来汇报。

**真正"像一个很专业的人"的魔力，恰恰在这两层的反差**：

> 表面上他跟你随便聊两句、很轻松、回复很短；
> 可一旦你需要真东西，他背后悄悄调动了数学家、画师、研究员……几条线同时跑，
> 然后回来跟你说一句"搞定了，结论是这个"。

现在的 MicroClaw 恰好两层都偏"机器人"：表层一口气甩长答案，深层虽有并发能力但闷头干完才出声。
下面分两层讲怎么改，最后缝成一个完整的人格。

---

## 第一层：怎么聊天才像人

### 1.1 默认简短（最重要的一条）

真人聊天 90% 的回复是 1–2 句话。长篇大论是例外，不是默认。

- **现状**：SOUL.md 写了 "direct and concise"，但实际行为仍倾向长答案；
  系统里也没有"长度预算"概念。
- **改法**：
  - SOUL.md 增加明确的**聊天风格**段：默认 1–3 句；**结论先行（BLUF）**；
    细节按需展开，先给答案再问"要展开吗？"。
  - 在 system prompt 注入**长度预算**：闲聊/确认类 ≤ 2 句；只有用户明确说"详细讲/写文档/给方案"
    才解锁长篇。
  - 落点：`SOUL.md` + `src/agent_engine.rs build_system_prompt`。

### 1.2 一次回多条消息（multi-bubble）

真人不会把一个想法塞进一坨，而是**拆成几条短消息连发**：
"等下" → "我看看" → "找到了，是这个 👇"。

- **现状**：机制其实**已经存在**——`send_message` 工具支持对话中途多次发送
  （`src/tools/send_message.rs:164` "Send a message mid-conversation"）。
  缺的是没人告诉模型"该这么做"。
- **改法**：
  - SOUL/prompt 鼓励：**把一个长回答拆成 2–3 条短消息**，而不是一条长的；
    先发一句"应一声/给结论"，再发细节。
  - 配**条间节奏**：每条之间 typing 心跳 + 300ms–2s 随机停顿，模拟"边想边发"
    （复用拟人化方案 C；Telegram/Discord 已有 typing 设施）。
  - 防刷屏：单回合气泡数设上限（如 ≤4），与进度推送共用节流原则。
  - 落点：`SOUL.md` + prompt + `send_message`（已具备）+ 频道层 typing/delay。

### 1.3 有节奏、会读空气

- **先应一声再答**：复杂问题先回"嗯，我看下"，再给结果（人就是这样，不会沉默 10 秒突然甩答案）。
- **一次只问一个问题**：不要列 1./2./3./4. 五连问，挑最关键的先问一个。
- **读情绪/语气**：用户急→简短直给；用户沮丧→先共情；用户轻松→可以皮一下。
  （= 拟人化方案 B：`<conversation_mood>` 动态注入，SOUL 管人格、mood 管此刻语气。）
- **知道什么时候不说话**：群聊里没被 @ 且没价值就不插嘴（= 拟人化方案 A 的 motivation 门控）。

### 1.4 像人的诚实

- 不知道就说"不确定，我查一下"，然后真去查——而不是一本正经地编。
- 报结果不报决心："搞定了"胜过"我试试"（SOUL 已有这条，保持）。

## 第二层：怎么做事才像一个专业的人

一个很专业的人，强不在"什么都自己会"，而在**知道把对的事交给对的专家，并同时盯着好几条线**。
这正好对应 MicroClaw 的 sub-agent 系统——把它从"一个通用打杂"升级成"一支专家团队"。

### 2.1 专科 Sub-Agent 团队（你提到的"数学、画画……各方面能力"）

**现状（关键发现）**：所有子代理现在是**同一个人**——
共用一段硬编码 system prompt（`src/tools/subagents.rs:328`，"You are a sub-agent assistant…"）
和同一套受限工具（`ToolRegistry::new_sub_agent`，`subagents.rs:320`）。
也就是说现在只有"一个通用临时工"，没有"专家"。

**改法：引入"专家档案（specialist profile）"——一张可扩展的注册表。**
每个 profile = 一套预设：

```
profile {
  name:          "mathematician"        // 专家名
  persona:       "你是严谨的数学家，逐步推导、显示过程、用代码验证"  // 专属 system prompt
  tools:         [code_exec, ...]        // 专属工具子集
  model:         可选（难题用更强模型）
  triggers:      ["算", "证明", "方程", "概率", ...]  // 路由关键词（可选）
}
```

**示例 roster（开箱即用 + 可无限扩展）：**

| 专家 | persona 要点 | 专属工具 |
|---|---|---|
| 🧮 mathematician | 逐步推导、代码验证、给出过程 | `bash`/code exec |
| 🎨 illustrator | 理解画面需求、迭代提示词 | `generate_image`, `describe_image` |
| 🔍 researcher | 多源检索、交叉验证、引用来源 | `web_search`, `web_fetch` |
| 💻 coder | 读写代码、跑测试、最小改动 | 文件工具, `bash` |
| ✍️ writer | 结构化写作、控制语气和长度 | 记忆/上下文 |
| 📊 analyst | 数据归类、统计、出图 | code exec, 文件工具 |

**主代理 = 会调度专家的全才**：
它自己保持简短、像人地聊天；遇到需要硬功夫的子问题，**悄悄**派给对的专家
（`sessions_spawn(profile="mathematician", label="证明收敛", task=...)`），
专家在后台跑，主代理继续陪你聊，完事了用一句话把结论端上来。

**可扩展性是设计目标**：新增一个专家 = 往注册表加一个 profile（配置/数据驱动），
**不改核心循环**。后续"音乐、翻译、法律、PPT……"都是加配置。

**落点（已定位）**：
- 把 `subagents.rs:328` 的硬编码 prompt → 改成按 `profile` 取 persona。
- 把 `ToolRegistry::new_sub_agent`（`subagents.rs:320`）→ 改成按 `profile` 取工具子集。
- `sessions_spawn` input schema 增加 `profile`（+ 上一份方案的 `label`）。
- profile 注册表放配置或 `src/tools/specialists.rs`（新），prompt 引导主代理选专家。

### 2.2 同时盯多条线 + 像同事一样汇报

这就是 `docs/concurrent-tasks-progress-reporting.md` 的全部内容，直接复用：
- 并发多任务（已具备）→ 给每条线起**人话名字**（label）；
- 运行中途**主动同步进展**（report_progress + 节流 relay）；
- 完成各自汇报，"现在都在忙啥"给**站会式清单**。

专家团队 + 进度汇报合起来，就是"一个背后有团队、还会主动 standup 的专业同事"。

### 2.3 专业素养（prompt/行为层）

- **结论先行（BLUF）**：先给答案/判断，再给依据；用户要才深挖。
- **知道边界**：超出能力或不确定，直说并给替代方案，不硬编。
- **记得你**：用现有三层记忆 + USER.md，聊天时自然带出"上次你提到的那个项目…"。
- **可靠 > 炫技**：SOUL 已有，保持。

## 第三层：把两层缝成一个完整的人格

一个完整的人 = **稳定的人格 + 流动的情绪 + 表层轻松 + 深层能干**，并且**按场景切换深浅**。

判定"该短答还是该认真深入"的几条简单规则（注入 prompt 即可）：

| 场景信号 | 行为 |
|---|---|
| 闲聊、确认、寒暄 | 1–2 句，可拆气泡，轻松 |
| 简单事实问题 | 一句话直给，必要时补一句来源 |
| 明确要"详细/方案/文档/教程" | 解锁长篇 + 结构化 |
| 需要硬功夫（算/画/查/写码/分析） | 后台派专家，主线继续聊，完事端结论 |
| 用户有情绪 | 先回应情绪，再处理事 |
| 群里没被 @ 且无价值 | 不说话 |

**人格稳定（SOUL）、情绪流动（mood）、深浅切换（场景判定）、能力深不可测（专家团队 + 并发）——
四者叠加，就是"一个很专业又很好聊的人"。**

## 一个"理想对话"剧本（把所有特性串起来演一遍）

```
用户：帮我看下这个积分收不收敛，顺便给我画个示意图，还有我们产品定价对不对得过竞品

Bot ：行，我分头看 👀                         ← 短·应一声
Bot ：积分和画图我让两条线同时跑，定价我查一下竞品   ← 一次多条·透明派活
      （后台悄悄：sessions_spawn 数学家[收敛性] / 画师[示意图] / 研究员[竞品定价]）

      ……（主代理继续陪聊，不阻塞）……

Bot ：📊 [竞品定价] 查了 3/5 家，咱们 $29 偏低，B 家同档 $49     ← 中途进度·节流
Bot ：🧮 [收敛性] 收敛，比值判别法，过程发你了                  ← 专家完成·一句话端结论
Bot ：🎨 [示意图] 画好了 👇 [图]                               ← 专家完成
Bot ：定价那条还在跑，剩两家。先说结论：你定价有空间往上提一点。  ← BLUF·简短

用户：竞品那个详细讲讲
Bot ：（解锁长篇，结构化展开五家对比……）                       ← 按需深入
```

短、分条、有节奏、读需求、按需深浅，背后却是一支专家团队几条线并发——这就是"像一个很专业的人"。

## 落地优先级（和已有两份文档对齐）

| 优先级 | 做什么 | 收益 | 成本 | 关联文档 |
|---|---|---|---|---|
| 🥇 P0 | **简短 + 多气泡聊天风格**（SOUL + prompt + 长度预算 + 节奏） | 最直接的"像人"感 | **低**（主要是 SOUL/prompt，机制已具备） | 本文 1.1–1.2 / 拟人化 C |
| 🥇 P0 | **情绪/语气动态** `<conversation_mood>` | "语气总算对了" | 低 | 拟人化 B |
| 🥈 P1 | **命名任务 + 站会一览** | 多线可感知、可查岗 | 中 | 并发方案 Phase 1 |
| 🥈 P1 | **专科 Sub-Agent 团队**（profile 注册表） | "背后有专家"的专业深度 | 中（改两处 + 加配置） | 本文 2.1 |
| 🥉 P2 | **运行中途进度推送** | 同事感 standup | 中 | 并发方案 Phase 2 |
| 🥉 P2 | **主动性 / 不被问也同步**（motivation 门控） | 真正"活"的感觉 | 中 | 拟人化 A / 并发方案 Phase 3 |

**建议开发顺序**：先 P0（一两处 SOUL/prompt 改动，立刻让它"像人地聊天"，零架构风险）→
再 P1（命名任务 + 专家团队，给它"专业的深度"）→ 最后 P2（主动性，让它"活"起来）。

---

## 一句话总结

> **像人 = 表层会聊天（短、分条、有节奏、读空气）+ 深层很能干（专家团队、多线并发、主动汇报），
> 并且懂得在两者间按场景切换。**
> MicroClaw 的底盘（记忆、并发、投递、子代理）已经够强，缺的是**让它表层放松下来像人地说话，
> 让它深层把"一个通用打杂"升级成"一支可扩展的专家团队"。**
