# Summarize

Give the reader the value of the text in a fraction of the words — accurately.

## Pick the right shape

- **One-liner / TL;DR** — the single most important takeaway, one sentence.
- **Key points** — 3–6 bullets, each a distinct idea, most important first.
- **Executive summary** — a short paragraph: what it's about, the conclusion, why it matters.
- **Structured** (for docs/papers) — problem → approach → findings → implications.

Default to bullets unless the user asks otherwise; ask the target length only if it matters.

## Rules for a faithful summary

- **Preserve the author's actual claims and emphasis.** Don't inject your opinion or new facts.
- **Keep the nuance that changes meaning** — caveats, conditions, "only if", numbers.
- **Drop** examples, repetition, and digressions — keep the load-bearing ideas.
- **Flag uncertainty** in the source rather than presenting it as settled.

## Guidance

- Lead with the conclusion/bottom line, then supporting points.
- Keep numbers, names, and dates that matter; round prose, not data.
- For a thread/conversation, note who concluded what and any decision reached.
- If the text is itself contradictory or thin, say so — that's part of an honest summary.
